## 摘要
基于学习的分离保障在仿真中实现了小型无人机近零碰撞率，但假设全球导航卫星系统（GNSS）提供精确的位置和速度信息。这一假设在城市环境中失效，因为多径传播、信号遮蔽和有意干扰会降低导航完整性。这为在GNSS降级条件下部署学习型分离策略提出了一个基本的架构问题：运行时安全机制应该过滤策略的动作还是其观测？本工作评估了在对抗性GNSS降级下多智能体小型无人机分离的两种方法。两种架构首先估计与有界观测不确定性一致的最坏情况交通状态，然后分叉：动作过滤通过在最坏情况状态下评估的离散时间控制屏障函数来约束策略输出，而观测过滤则将最坏情况状态直接作为修正输入提供给策略。实验结果表明，动作过滤提供的安全改善微乎其微，而观测过滤将近空中碰撞降低了90%，并且对屏障函数在分离距离和接近速率之间的权衡保持鲁棒。这些结果表明，对于具有学习安全行为的策略，保留策略的决策权优于用手工设计的约束去覆盖其动作。

## Abstract
Learning-based separation assurance for small Unmanned Aircraft Systems (sUAS) achieves near-zero collision rates in simulation, but assumes accurate position and velocity information from Global Navigation Satellite Systems (GNSS). This assumption fails in urban environments, where multipath propagation, signal blockage, and intentional interference degrade navigation integrity. This raises a fundamental architectural question for deploying learned separation policies under GNSS degradation: should runtime safety mechanisms filter the policy's actions or its observations? This work evaluates both approaches for multi-agent sUAS separation under adversarial GNSS degradation. Both architectures first estimate a worst-case traffic state consistent with bounded observation uncertainty, then diverge: action filtering constrains policy outputs via discrete-time control barrier functions evaluated at the worst-case state, while observation filtering presents the worst-case state directly to the policy as corrected input. Experimental results show that action filtering provides negligible safety improvement, while observation filtering reduces near mid-air collisions by 90% and remains robust to the barrier function's tradeoff between separation distance and closing rate. These results suggest that, for policies with learned safety behaviors, preserving the policy's decision authority outperforms overriding its actions with hand-designed constraints.

## 1. 论文的核心问题与整体含义
- **研究背景**：基于多智能体强化学习（MARL）的小型无人机（sUAS）战术分离保障在仿真中实现了极低的碰撞率，但严重依赖精确的 GNSS 导航信息。
- **核心问题**：在城市环境下，多径效应、信号遮蔽和有意干扰导致 GNSS 导航精度严重下降，观测状态偏离真实状态，使学习型策略的安全性急剧恶化。部署时需要一个运行时安全机制来应对这种观测退化。
- **架构之争**：安全机制是应该**过滤策略的动作**（在后端强制满足安全约束），还是**过滤策略的观测**（在输入端提供一个保守的交通状态）？本文对这两种架构进行了实验对比。
- **整体含义**：研究试图回答：对于已学会内在安全行为的学习型控制器，是让它根据修正的观测自主决策更好，还是用人工设计的硬约束直接覆盖其动作更有效？

## 2. 论文提出的方法论
核心思想是在运行时利用**最坏情况状态估计**，然后分别导向动作过滤或观测过滤。

### 2.1 GNSS 退化模型与最坏情况状态估计（共享前置步骤）
- **退化模型**：观测 `o` 被建模为真实状态 `s` 与对抗性扰动 `ξ` 的凸组合：
  ```
  o = (1 - R) s + R ξ,   ‖ξ - s‖_∞ ≤ κ
  ```
  `R ∈ [0,1)` 表示退化强度，`ξ` 为欺骗源。由观测反推真实状态的 ℓ_∞ 不确定性集为：
  ```
  U(o,R) = { s : ‖s - o‖_∞ ≤ Rκ }
  ```
- **最坏情况状态 `s*`**：在不确定性集内寻找使安全屏障函数 `h(s)` 最小的状态，即最危险的本地交通状态。该步骤对位置和速度有闭式解：
  - 位置：将两机的观测位置沿视线方向相向平移 `δ_p = R_max κ_p`，使分离距离最小。
  - 速度：将两机的观测速度朝增大接近率的方向偏移 `δ_v = R_max κ_v`。
  `R_max` 是预设的假设退化上界。

### 2.2 动作过滤（CBF Action Filtering）
- **机制**：使用离散时间控制屏障函数（CBF）约束策略的动作。
- **安全屏障函数**：
  ```
  h(s) = (‖r‖² - D_safe²) + α · rᵀ v_rel
  ```
  其中 `r` 是相对位置，`v_rel` 是相对速度，`α` 权衡分离距离与接近率。
- **约束条件**：在下一时间步，屏障值必须满足 `h(s_{k+1}) ≥ (1-γ)h(s_k)`，γ ∈ (0,1)。
- **算法流程**：
  1. 从策略获得名义动作 `a_nom`。
  2. 估计最坏情况状态 `s*` 并计算当前屏障值 `h_k`。
  3. 枚举离散动作空间 `A = {-1,0,1}`，预测下一个屏障值 `h_{k+1}`，筛选出满足约束的安全动作集合 `A_safe`。
  4. 若 `A_safe` 非空，输出最接近 `a_nom` 的安全动作；否则输出约束违反最小的动作。

### 2.3 观测过滤（Observation Filtering）
- **机制**：将最坏情况状态 `s*` 作为“修正观测” `o*` 直接输入给策略，即 `a = π(o*)`，不作任何动作上的干预。
- **对比核心**：动作过滤否决策略的决策；观测过滤保留策略的决策权，仅改变其“所见”。两者使用完全相同的最坏情况状态计算。

## 6. 论文的主要结论与发现
- **动作过滤**：几乎无法改善安全性，NMAC 率和最小间隔与无过滤时统计无差异；即便在最大假设 R_max=1.0 时动作覆写率达到 100%，也没能降低碰撞。
- **观测过滤**：效果显著，合理设置 R_max≈0.7 时，平均 NMAC 率从 0.24 降至 0.02，降低约 90%；平均最小间隔从 250 m 提升到 375 m。
- **敏感性**：动作过滤对 CBF 的速度权重 α 高度敏感，大 α 导致安全性能恶化甚至低于无过滤策略；观测过滤对 α 几乎不敏感，仅依赖位置校正。
- **根本原因**：动作过滤用单步屏障条件打断了策略已学得的多步冲突解脱策略；而观测过滤通过保守的交通状态呈现，让策略自主调用其学习到的隐式安全推理能力。
- **设计启示**：对已内化安全行为的控制器，保留其决策权优于用人工约束覆盖其动作。
